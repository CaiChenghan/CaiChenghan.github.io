子模块在Xcode上的使用

git子模块适用于iOS项目，但由于iOS项目不只是简单的文件读取，需要 Target 引用，故子模块的 修改 、 提交 需在 主工程 中进行。如此，则引用关系会跟随主项目修改提交，一并上传到远程服务器，其他用户下拉的代码，则包含子模块的引用关系。否则，只是简单的文件引入而没有引用关系，会导致子模块在项目中无法使用。
